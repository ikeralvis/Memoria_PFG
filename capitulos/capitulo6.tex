\chapter{Presupuesto y evaluación de costes}

Este capítulo presenta la evaluación económica de la plataforma TankGo bajo dos enfoques complementarios:
el \textbf{coste de desarrollo teórico}, que representa la valoración en el mercado del esfuerzo de ingeniería invertido,
y el \textbf{coste de explotación real}, derivado del despliegue y mantenimiento de la infraestructura en producción.
El análisis demuestra la viabilidad financiera del proyecto y la optimización de costes lograda mediante la adopción
de paradigmas \textit{serverless} y arquitecturas elásticas de pago por uso.

\section{Costes laborales}

a perfiles de Ingeniería de Software Junior con competencias transversales en desarrollo web, arquitecturas cloud y pipelines de

\section{Material, software e infraestructura}
% ------------------------------------------------------------

El único elemento hardware empleado ha sido el ordenador personal del autor, valorado en
1.200~€. Aplicando amortización lineal a 4 años con un uso dedicado al proyecto del 60\,\%
durante 9 meses, la amortización imputable es de \textbf{135~€}.

Las herramientas de software empleadas (entornos de ejecución Python y Node.js, librerías de cliente,
orquestadores Docker Desktop, editores Visual Studio Code y los sistemas de control de versiones Git/GitHub)
se rigen por licencias de código abierto o modalidades comunitarias sin coste, representando una partida de \textbf{0~€}.

En lo que respecta a la infraestructura en la nube durante la fase de desarrollo, los servicios distribuidos
(Google Cloud Run, Cloud Storage, Cloud Build, Artifact Registry, Neon PostgreSQL y las llamadas de testeo en Google AI Studio)
se mantuvieron dentro de las capas de uso gratuito (\textit{free tier}) o se financiaron mediante créditos académicos de vigencia limitada. Sin embargo una vez pasada la fecha,
es posible que se hayan incurrido en costes menores, aunque no se dispone de un desglose detallado. Para reflejar el coste real durante el desarrollo,
se ha considerado únicamente el gasto efectivo del registro del dominio \texttt{tankgo.dev}, que ascendió a \textbf{11~€}. Este registro se realizó a través de Cloudflare,
que ofrece un precio competitivo y una gestión sencilla de DNS, y se ha mantenido activo durante todo el periodo de desarrollo.

proyecta una estimación de los costes recurrentes de infraestructura bajo un escenario operativo real.
El modelo asume una carga mensual moderada estabilizada en 500.000 peticiones agregadas al API Gateway,
prescindiendo de créditos promocionales y aplicando las tarifas completas de los proveedores cloud.

TankGo podría operar como servicio real con un coste de infraestructura inferior a
\textbf{1.000~€/año}, lo que refleja la eficiencia económica de las arquitecturas serverless
y los modelos \textit{pay-as-you-go} para proyectos de escala media, y confirma la viabilidad
económica de la solución más allá del ámbito académico.

Como modelo de negocio potencial, TankGo podría adoptar una estrategia
\textit{freemium}: acceso gratuito para usuarios individuales con funcionalidades
básicas (consulta de precios, búsqueda por proximidad y recomendaciones de
ruta) y una suscripción de pago mensual que desbloquee capacidades avanzadas,
como un mayor número de estaciones favoritas, historial extendido de precios,
predicciones personalizadas y notificaciones de variación. Este modelo permite
captar base de usuarios sin fricción inicial mientras monetiza a los usuarios
con mayor engagement, un patrón consolidado en aplicaciones de movilidad y
utilidad cotidiana.
